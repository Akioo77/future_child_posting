# Future Child Posting — 项目规格说明

> 最后更新：2026-07-01

---

## 🎯 项目概述

**项目名称**：Future Child Posting
**类型**：研究项目 — AI 辅助家长在社交媒体分享孩子内容时的隐私风险检测工具
**目标用户**：有孩子的父母（所有年龄段）
**核心功能**：用户上传图片 + 输入文字 → AI 检测隐私风险 → 输出风险报告 + 修改建议

---

## 📋 研究问题（RQ）

> RQ：AI 辅助的「发布前风险扫描」能否有效帮助父母识别和规避分享孩子内容时的隐私和伦理风险？

**产出**：原型 + 用户研究 → 论文（CHI/CSCW/IDC）

---


## 🔬 功能范围（Phase 1 MVP）

### 核心功能
1. **图片上传** — 用户上传 1 张图片（支持 JPG/PNG，最大 10MB）
2. **文字输入** — 用户输入想发布的文字说明（可选）
3. **AI 风险分析** — 调用多模态大模型，联合分析图片+文字
4. **风险报告展示** — 列出检测到的所有风险点（类型+等级+说明）
5. **建议呈现** — 对每个风险给出修改建议

### 不做（Phase 1）
- ❌ 实际图片脱敏/编辑
- ❌ 用户登录系统
- ❌ 历史记录存储
- ❌ 多图片批量分析

---

## 🔍 风险检测类型（5类，来自 Imago Obscura）

| ID | 风险类型 | 说明 | 示例 |
|----|---------|------|------|
| R1 | **自我披露风险** | 意外暴露个人习惯/健康/生活事件 | 生日蜡烛暴露年龄、药瓶暴露健康 |
| R2 | **身份暴露风险** | 可见身份信息被识别 | 纹身、虹膜、全名、ID卡 |
| R3 | **机密信息泄露** | 背景中的商业/私人敏感信息 | 白板项目计划、电脑屏幕、文件 |
| R4 | **位置暴露风险** | 可识别地理位置的信息 | 窗外地标、学校标志、门牌 |
| R5 | **旁观者风险** | 背景中非自愿被拍摄的路人 | 街上路人、其他孩子 |

**风险等级**：高（H）/ 中（M）/ 低（L）

## 🛠️ 技术栈

### 后端
- **框架**：FastAPI（Python 3.10+）
- **AI**：Minimax VL API（多模态分析）
- **端口**：8000

### 前端
- **框架**：Next.js 14（App Router，TypeScript）
- **样式**：Tailwind CSS
- **端口**：3000

### 开发规范
- 后端 API：`RESTful JSON`
- 前后端通信：`fetch` / `axios`
- 不需要数据库（Phase 1）

## 🔌 API 设计

### POST /analyze

**请求**：
```json
{
  "image": "data:image/jpeg;base64,/9j/4AAQ...",  // Base64 编码图片
  "text": "今天带儿子去XX小学报名了，真开心！"   // 可选，空字符串表示无
}
```

**响应**：
```json
{
  "risks": [
    {
      "id": "R4",
      "type": "位置暴露风险",
      "level": "H",
      "description": "学校名称 XX 小学 可定位孩子就读学校",
      "source": "text"
    },
    {
      "id": "R1",
      "type": "自我披露风险",
      "level": "M",
      "description": "孩子穿着校服出现在图片中",
      "source": "image"
    }
  ],
  "suggestions": [
    "删除文字中的学校名称，或用 'XX 学校' 代替",
    "考虑给校服打码或换一张不露校服的图片"
  ]
}
```

**错误响应**：
```json
{
  "error": "图片大小超过 10MB 限制"
}
```

---

## ⚠️ 已知限制

1. **API 额度**：Minimax VL 有免费额度限制，Phase 1 够用
2. **图片大小**：最大 10MB（API 限制）
3. **准确性**：AI 分析有一定误判率，用户研究时需注意
4. **Proxy**：需要通过代理（7897端口）访问 Minimax API
